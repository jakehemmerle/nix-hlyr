# nix-hlyr

Unofficial [Nix](https://nixos.org) flake for the **HumanLayer / Riptide** desktop app — the AI coding agent powered by Claude ([humanlayer.dev](https://humanlayer.dev/)).

Upstream ships the app only as a notarized macOS DMG through a [Homebrew cask](https://github.com/humanlayer/homebrew-humanlayer). This repo repackages that exact artifact for Nix and keeps it current automatically: a scheduled GitHub Action reads the upstream cask and commits any version bump to `main` (see [Auto-update](#auto-update)).

> **Platform:** Apple Silicon only (`aarch64-darwin`), matching upstream — the DMG is arm64-only.
>
> **Unofficial:** not affiliated with or endorsed by HumanLayer. It fetches the official signed `Riptide-darwin-arm64.dmg` release, verifies its checksum, and installs the notarized `HumanLayer.app` unmodified (the Developer ID signature is preserved). The app also bundles the `riptided` daemon, which is linked onto `PATH`.

## Quick start

Open the app without installing it:

```sh
nix run github:jakehemmerle/nix-hlyr
```

Run the bundled daemon directly:

```sh
nix run github:jakehemmerle/nix-hlyr#riptided
```

Install into your profile:

```sh
nix profile install github:jakehemmerle/nix-hlyr
```

The build result contains `Applications/HumanLayer.app` and `bin/riptided`. Note that `nix profile install` puts `riptided` on your `PATH` but places the `.app` under `~/.nix-profile/Applications`, which macOS Spotlight/Finder do **not** index — use `nix run` for the GUI, or the nix-darwin / Home Manager integrations below (which link the app into a discoverable folder).

## Use it in your config

In each example, the `inputs.nix-hlyr` line belongs in your **flake's** `inputs` block, and the rest goes in the relevant module. The module needs access to your flake `inputs` — pass them through with `specialArgs = { inherit inputs; }` (nix-darwin) or `extraSpecialArgs = { inherit inputs; }` (Home Manager) where you build the configuration.

### nix-darwin

```nix
# flake.nix
inputs.nix-hlyr.url = "github:jakehemmerle/nix-hlyr";

# a nix-darwin module (with `inputs` in specialArgs):
{ inputs, ... }:
{
  environment.systemPackages = [ inputs.nix-hlyr.packages.aarch64-darwin.humanlayer ];
}
```

`.app` bundles from a package show up under **`/Applications/Nix Apps`** on nix-darwin (Spotlight/Finder can launch them).

### Home Manager

```nix
# flake.nix
inputs.nix-hlyr.url = "github:jakehemmerle/nix-hlyr";

# a Home Manager module (with `inputs` in extraSpecialArgs):
{ inputs, ... }:
{
  home.packages = [ inputs.nix-hlyr.packages.aarch64-darwin.humanlayer ];
}
```

Home Manager links `.app` bundles into `~/Applications/Home Manager Apps`.

### Overlay

To get `pkgs.humanlayer` everywhere, apply the overlay in your nixpkgs config:

```nix
{ inputs, ... }:
{
  nixpkgs.overlays = [ inputs.nix-hlyr.overlays.default ];
}
```

## Flake outputs

| Output | Description |
| --- | --- |
| `packages.aarch64-darwin.humanlayer` (`default`) | The app + bundled `riptided` binary |
| `apps.aarch64-darwin.default` | `nix run` → opens `HumanLayer.app` |
| `apps.aarch64-darwin.riptided` | `nix run .#riptided` → runs the daemon |
| `overlays.default` | Adds `humanlayer` to nixpkgs |
| `checks.aarch64-darwin.build` | Builds the package (`nix flake check`) |

## Auto-update

The upstream `humanlayer` cask is regenerated by HumanLayer's own release pipeline, so it is an always-current source of truth for `{version, url, sha256}`. This repo tracks it:

1. [`.github/workflows/update.yml`](.github/workflows/update.yml) runs every 15 minutes (and on demand).
2. [`scripts/update.sh`](scripts/update.sh) fetches [`Casks/humanlayer.rb`](https://github.com/humanlayer/homebrew-humanlayer/blob/main/Casks/humanlayer.rb), extracts the version / URL / checksum, and converts the checksum to SRI, writing [`pkgs/humanlayer/source.json`](pkgs/humanlayer/source.json).
3. If anything changed, the workflow builds the new version on an Apple Silicon runner **and** runs [`scripts/verify-signature.sh`](scripts/verify-signature.sh), which fails the bump unless the app is Apple-notarized and signed by HumanLayer's Developer ID (Team ID `89C6S2SYU3`). Only then does it commit `chore: bump humanlayer to <version>` directly to `main`.

`pkgs/humanlayer/default.nix` reads `source.json`, so a version bump never touches Nix code. To update by hand:

```sh
./scripts/update.sh && nix build .#humanlayer && ./scripts/verify-signature.sh result
```

> **Note:** GitHub disables scheduled workflows after 60 days with no repository activity. HumanLayer releases far more often than that, so in practice the auto-commits keep it alive — but if upstream ever paused for 60+ days, GitHub would email you and you'd re-enable `update.yml` from the Actions tab with one click.
